## Module Design 模組設計

### 1. `analysis.py` — 主控腳本
- 進入點: `main()`
- 掃描 `_daily_input/` 所有 `.txt` 檔案
- 呼叫 Claude API 產出綜合分析
- 對每個議題啟動深度分析（可並行）
- 完成後觸發歸檔

### 2. `config.py` — 設定管理
- 從 `.env` 讀取 API 金鑰
- 定義資料夾路徑常數
- 定義報告格式模板

### 3. `claude_client.py` — API 封裝
- 封裝 `anthropic.Anthropic` 客戶端
- 自動重試機制（3 次, 指數退避）
- 使用量追蹤與日誌

### 4. `report_writer.py` — 報告產出
- Markdown 報告生成
- 資料夾自動建立
- Chicago 格式引用文獻格式化

### 5. `archiver.py` — 歸檔管理
- 檔案移動與重命名
- 索引維護
- 重複偵測

## Technology Decisions 技術決策

| 決策 | 選擇 | 理由 |
|------|------|------|
| AI 模型 | claude-opus-4-1 | 深度分析品質最佳 |
| 語言 | Python 3.10+ | 生態系豐富，anthropic SDK 原生支援 |
| 設定管理 | python-dotenv | 簡單安全的環境變數管理 |
| 排程 | Grok + Claude Code Skill | 現有流程，低維護成本 |
| 版本控制 | Git + GitHub | 已建立 garden94030 帳號 |

## Error Handling 錯誤處理策略

1. **API 失敗**: 重試 3 次 → 記錄錯誤 → 跳過該檔案 → 繼續處理其他檔案
2. **檔案讀取失敗**: 記錄錯誤 → 不歸檔 → 通知使用者
3. **輸出寫入失敗**: 檢查磁碟空間 → 記錄錯誤 → 保留原始輸入
