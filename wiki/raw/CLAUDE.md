# 🗃️ 原始文獻圖書館 (Raw Database)

這裡是 20 人團隊的「未加工資源」集散地。
不論文章排版多混亂、程式碼多長，只要覺得有價值，請放心大膽地**先存再說**。

## 📥 這個資料夾的作用？
- 這是對接 **Obsidian Web Clipper** 或其他瀏覽器擴充外掛的最佳預設儲存位置。
- 分析師平時爬梳網路時，若看到對情報分析有潛力價值的文章、原始論文、Twitter (X) 長文、智庫報告 PDF、或任何參考用截圖，都可以不動腦筋直接 Dump 進這裡。

## 🤖 給 Claude Code (分析師助理) 的指令
當任何團隊成員請你 (Claude) 協助整理情報時，你可以：
1. **來這裡尋寶**：當團隊提到某個概念（例如：「幫我查一下前天的外媒報導」），你可以進來 `raw/` 資料夾中進行地毯式搜索。
2. **提煉並轉移**：當你分析完 `raw/` 裡的一篇長文，並將其有價值的部分萃取到 `wiki/concepts/` 中之後，建議在原始長文的最上面加上 `> [!note] 已摘錄至 [[某某概念]]` 以利追蹤。

## ⚠️ 注意事項

- 這裡**不追求排版與格式檢查**。
- `wiki_compiler.py` (自動生成的後勤系統) 預設**不會**去動這裡的資料。這裡是專屬於人類分析師與他們的在地 Claude 隨身助理的游擊戰場。

## 🐛 已知問題：Obsidian Web Clipper 檔名含斜線

**問題**：當網頁標題含有 `/` 字元（例如：「(2) 首頁 / X」），Web Clipper 無法儲存，因為 `/` 在 Windows 為非法檔名字元。

**解決方法**（三選一）：

1. **手動改標題**（推薦）：在 Web Clipper 的 "Title" 欄位，存檔前先把 `/` 改為 `-` 或刪除。例如：`首頁 - X`
2. **使用 Clipper 自訂模板**：在 Web Clipper 設定中，將 `{{title}}` 改為 `{{title | replace("/", "-")}}`
3. **存為 .txt 轉存**：直接複製頁面文字，貼成 .txt 放進 `_daily_input/` 讓系統自動處理
